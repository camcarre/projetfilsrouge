<objective>
## Goal
Ajouter un **backtest chiffré** (évaluation out-of-sample) au pipeline de prédiction pour prouver la compétence 5 de la grille (ML + évaluation), qui passe de C à B.

## Purpose
Aujourd'hui `GET /api/predict/stock` produit un forecast (HF LLM → ETS → régression linéaire) mais **sans aucune métrique d'évaluation ni split train/test**. On ajoute une évaluation honnête : on cache les K derniers points connus, on entraîne le modèle sur le reste, on prédit K pas, on compare à la vérité terrain → **RMSE, MAE, R²**.

## Output
- `prediction.backtest = { horizon, train_size, test_size, rmse, mae, r2, model }` dans la réponse de l'endpoint.
- Une carte « Fiabilité du modèle (backtest) » sur la page Analyse affichant RMSE / MAE / R².
- Un test pytest qui vérifie le calcul des métriques.
</objective>

<acceptance_criteria>
## AC-1 : split train/test réel
Given un historique de N cours (N ≥ 30) When on appelle le backtest Then les K derniers points (test) ne servent JAMAIS à l'entraînement, et le modèle est fit uniquement sur les N−K premiers (train).

## AC-2 : métriques out-of-sample exposées
Given un backtest calculé When l'endpoint répond Then `prediction.backtest` contient `rmse`, `mae`, `r2` (floats), `horizon`, `train_size`, `test_size`, `model`, tous cohérents (rmse ≥ 0, mae ≥ 0, r2 ≤ 1).

## AC-3 : forecast existant intact
Given l'endpoint actuel When on ajoute le backtest Then le forecast (`prediction.forecast`, `predicted_price`, `history`, `confidence`, `model`) reste identique en structure — aucune régression.

## AC-4 : affichage UI
Given une prédiction avec backtest When l'utilisateur ouvre la page Analyse Then une carte affiche RMSE, MAE et R² lisiblement (tabular-nums), avec un état vide si `backtest` absent.

## AC-5 : test automatisé
Given la fonction de backtest When on lance pytest Then un test vérifie sur une série connue que rmse/mae/r2 sont calculés correctement (valeurs attendues sur un cas déterministe).

<task type="auto">
  <name>Fonction de backtest dans analytics_service</name>
  <files>backend/services/analytics_service.py</files>
  <action>
Ajouter `def backtest_forecast(prices: list[float], horizon: int | None = None) -> dict`:
- horizon par défaut = `min(15, max(5, len(prices) // 5))` ; garder au moins 20 points de train, sinon `ValueError` (frontière STD-002).
- `train = prices[:-K]`, `test = prices[-K:]`.
- Fit ETS `ExponentialSmoothing(train, trend='add', initialization_method='estimated').fit(optimized=True)` → `preds = ets.forecast(K)`. Sur exception ETS → fallback régression linéaire numpy (`np.polyfit(x, y, 1)` sur train, extrapolée sur K), `model = "linear-regression"`, sinon `model = "ETS"`.
- Métriques out-of-sample avec numpy : `rmse = sqrt(mean((preds-test)**2))`, `mae = mean(abs(preds-test))`, `r2 = 1 - ss_res/ss_tot` (avec `ss_tot = sum((test-mean(test))**2)` ; si `ss_tot == 0` → `r2 = 0.0` pour éviter la division par zéro).
- Retourner `{ "horizon": K, "train_size": len(train), "test_size": len(test), "rmse": round(float(rmse), 4), "mae": round(float(mae), 4), "r2": round(float(r2), 4), "model": model }`.
- Type hints obligatoires (rules/python.md). Pas de `print` (logger si besoin, sinon rien).
