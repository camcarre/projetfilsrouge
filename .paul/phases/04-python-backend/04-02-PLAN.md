<task type="auto">
  <name>Task 2 : Routes Transactions (BUY/SELL atomique)</name>
  <files>backend/main.py</files>
  <action>
    **Modèle Pydantic** :
    ```python
    class TransactionBody(BaseModel):
        symbol: str
        type: str  # "BUY" | "SELL"
        quantity: float
        price: float
        portfolioId: Optional[str] = "default"
        date: Optional[str] = None
    ```

    **GET /api/transactions** :
    - Vérifier auth, 401 si absent
    - SELECT * FROM transactions WHERE user_id=? ORDER BY date DESC
    - Retourner `{ transactions: [...] }`

    **POST /api/transactions** :
    - Vérifier auth, 401 si absent
    - Valider symbol, type, quantity, price requis → 400 si manquant
    - Logique atomique (SQLite ne supporte pas `BEGIN` imbriqué, utiliser isolation_level=None et BEGIN/COMMIT/ROLLBACK explicites) :
      1. INSERT INTO transactions
      2. SELECT existing asset WHERE user_id=? AND symbol=?
      3. Si BUY et actif existant : newQty = old + qty, newPRU = (oldQty × oldPRU + qty × price) / newQty → UPDATE
      4. Si BUY et pas d'actif : INSERT INTO assets (category='action' par défaut)
      5. Si SELL et actif existant : newQty = old - qty
         - Si newQty < 0 : ROLLBACK, retourner 400 "Quantité insuffisante"
         - Si newQty = 0 : DELETE FROM assets
         - Si newQty > 0 : UPDATE quantity
      6. COMMIT
    - Retourner 201 `{ id: transactionId, message: "Transaction enregistrée" }`

    Note : Python sqlite3 gère les transactions via `conn.execute("BEGIN")` + `conn.commit()` / `conn.rollback()` — ne pas utiliser `db.transaction()` (API JS inexistante).
  </action>
