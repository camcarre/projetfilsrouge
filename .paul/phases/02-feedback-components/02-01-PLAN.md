<task type="auto">
  <name>Task 3 : Page transition fade-in 150ms</name>
  <files>src/index.css, src/components/ui/PageTransition.tsx, src/App.tsx</files>
  <action>
    **Étape A — index.css** : Ajouter dans `@layer utilities` (après `.tabular-nums`) :
    ```css
    .page-enter {
      animation: fadeIn 150ms ease-out both;
    }

    @keyframes fadeIn {
      from { opacity: 0; transform: translateY(4px); }
      to   { opacity: 1; transform: translateY(0); }
    }
    ```

    **Étape B — PageTransition.tsx** : Créer `src/components/ui/PageTransition.tsx` :
    ```tsx
    import type { ComponentChildren } from 'preact'
    import { useLocation } from 'react-router-dom'

    interface PageTransitionProps {
      children: ComponentChildren
    }

    export function PageTransition({ children }: PageTransitionProps) {
      const location = useLocation()
      return (
        <div key={location.pathname} className="page-enter">
          {children}
        </div>
      )
    }
    ```
    Le `key={location.pathname}` force le remontage du div à chaque navigation, relançant l'animation CSS.

    **Étape C — App.tsx** : Wrapper les `<Routes>` avec `<PageTransition>` :
    ```tsx
    import { PageTransition } from '@/components/ui/PageTransition'
    // ...
    <Suspense fallback={<PageLoadingFallback />}>
      <PageTransition>
        <Routes>
          {/* routes inchangées */}
        </Routes>
      </PageTransition>
    </Suspense>
    ```
    Ne pas toucher aux routes elles-mêmes, ni à ErrorBoundary, ni à Layout.
  </action>
  <verify>Ouvrir http://localhost:5174, naviguer entre pages — chaque page apparaît avec un léger fondu/glissement de 150ms. `npm run build` passe sans erreur.</verify>
  <done>AC-3 satisfait : fade-in 150ms sur chaque navigation, implémenté en CSS pur sans librairie d'animation</done>
