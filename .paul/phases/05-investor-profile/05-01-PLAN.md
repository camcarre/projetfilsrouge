<objective>
## Goal
Ajouter la table `investor_profiles` et les endpoints backend (GET/POST /api/profile + GET /api/etfs/recommended) pour stocker le profil investisseur et scorer les ETFs.

## Purpose
Sans profil stocké côté serveur, le moteur de recommandation ETF ne peut pas fonctionner. Ce plan pose les fondations backend du système de recommandation — cœur du cahier des charges.

## Output
- Table `investor_profiles` dans SQLite
- `GET /api/profile` — retourne le profil de l'utilisateur connecté
- `POST /api/profile` — crée ou met à jour le profil
- `GET /api/etfs/recommended` — retourne les ETFs scorés selon le profil
</objective>

    ```sql
    CREATE TABLE IF NOT EXISTS investor_profiles (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        user_id INTEGER UNIQUE NOT NULL,
        risk_tolerance INTEGER NOT NULL CHECK(risk_tolerance BETWEEN 1 AND 5),
        investment_horizon TEXT NOT NULL CHECK(investment_horizon IN ('short', 'medium', 'long')),
        investment_goal TEXT NOT NULL CHECK(investment_goal IN ('growth', 'income', 'preservation')),
        monthly_investment REAL DEFAULT 0,

        created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
        updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
        FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
    );
    ```

<task type="auto">
  <name>Task 3: Endpoint GET /api/etfs/recommended avec scoring</name>
  <files>backend/main.py</files>
  <action>
    Ajouter après /api/profile :

    GET /api/etfs/recommended :
    - Récupérer user via get_current_user(authorization)
    - Récupérer profil → si absent: HTTPException(404, {"error": "profile_required"})
    - Appeler get_etfs() du yahoo_etf_service (liste complète sans filtres)
    - Pour chaque ETF, calculer match_score (0-100) :

    Algorithme de scoring (total 100 pts) :
    ```
    score_risk     = 30 pts  → comparaison risk_tolerance (1-5) vs volatilité ETF (perf1y)
    score_horizon  = 25 pts  → short → TER bas (<0.2) + ESG peu important ; long → perf1y prime
    score_esg      = 20 pts  → si esg_preference=True : AAA=20, AA=16, A=12, B=6 ; sinon 20 pts flat
    score_ter      = 15 pts  → TER <= 0.15 → 15pts ; <= 0.25 → 10pts ; <= 0.35 → 5pts ; > 0.35 → 0
    score_goal     = 10 pts  → growth → perf1y > 10% → 10pts ; income → ESG score élevé → 10pts ; preservation → TER bas → 10pts
    ```

    - Ajouter `match_score` à chaque ETF dict
    - Trier par match_score décroissant
    - Retourner {"etfs": [...], "profile_used": {risk_tolerance, investment_horizon, ...}}

    Utiliser round(score, 1) pour le match_score.
    Pas de dépendance externe nouvelle — calcul en Python pur.
  </action>
  <verify>curl -s http://localhost:8000/api/etfs/recommended -H "Authorization: Bearer VALID_TOKEN" | python3 -c "import sys,json; d=json.load(sys.stdin); print('OK scores:', [e['match_score'] for e in d['etfs'][:3]])"</verify>
