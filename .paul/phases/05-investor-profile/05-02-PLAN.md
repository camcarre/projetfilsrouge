<objective>
## Goal
Implémenter le questionnaire profil investisseur multi-étapes, le Redux slice profil, et brancher la page ETF sur le vrai endpoint de recommandation scoré.

## Purpose
L'interface du questionnaire transforme le backend du Plan 01 en feature utilisable. Sans ce plan, le profil ne peut pas être renseigné et les scores ETF restent du mock.

## Output
- `profileSlice.ts` — état Redux du profil investisseur
- `profileService.ts` — GET/POST /api/profile + GET /api/etfs/recommended
- `ProfileQuestionnairePage.tsx` — formulaire 5 étapes
- Route `/profile/questionnaire` dans App.tsx
- Redirect post-login vers questionnaire si profil absent
- ETF page utilise /api/etfs/recommended quand connecté
</objective>

## AC-1: Redux profileSlice
```gherkin
Given l'app démarre
When le store Redux est initialisé
Then profileSlice expose { profile: InvestorProfile | null, isLoading, error } et les actions setProfile, clearProfile, setLoading
```

## AC-2: profileService
```gherkin
Given l'utilisateur est connecté
When fetchProfile() est appelé
Then GET /api/profile est appelé avec le Bearer token, retourne le profil ou null si 404

Given l'utilisateur remplit le questionnaire
When saveProfile(data) est appelé
Then POST /api/profile est appelé avec les données, retourne le profil sauvegardé

Given l'utilisateur est connecté avec un profil
When fetchRecommendedEtfs() est appelé
Then GET /api/etfs/recommended est appelé, retourne les ETFs avec match_score réel
```

## AC-3: Questionnaire multi-étapes
```gherkin
Given l'utilisateur accède à /profile/questionnaire
When il navigue entre les étapes
Then 5 étapes s'affichent séquentiellement : (1) tolérance risque, (2) horizon, (3) objectif, (4) montant mensuel, (5) préférence ESG + niveau
Et il peut naviguer Précédent/Suivant
Et la soumission finale appelle saveProfile et redirige vers /etf

Given l'utilisateur soumet une étape incomplète
When il clique Suivant
Then la validation bloque l'avancement avec un message d'erreur
```

## AC-4: Redirect post-login
```gherkin
Given un utilisateur qui vient de se connecter sans profil existant
When AuthPage redirige après login réussi
Then l'app fetch le profil, si absent redirige vers /profile/questionnaire, sinon vers /
```

## AC-5: ETF page branchée sur scoring réel
```gherkin
Given un utilisateur connecté avec un profil
When la page ETF se charge
Then fetchRecommendedEtfs() est appelé et les ETFs affichent le match_score réel (pas mock)

Given un utilisateur non connecté ou sans profil
When la page ETF se charge
Then fallback sur fetchEtfs() standard avec match mock
```
