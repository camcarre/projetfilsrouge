  <action>
    Nouveau fichier. Importer yfinance, pandas, numpy, pandas_ta (try/except pour import optionnel).

    Fonction `get_indicators(ticker: str, period: str = "3mo") -> dict` :
    - Télécharger OHLCV avec `yf.Ticker(ticker).history(period=period)`
    - Si df vide ou < 20 lignes : lever ValueError("Données insuffisantes")
    - Si pandas_ta disponible : df.ta.rsi(length=14), df.ta.macd(), df.ta.bbands()
    - Si pandas_ta non disponible : calculer RSI manuellement (numpy ewm), MACD = EMA12 - EMA26, Bollinger = SMA20 ± 2*std
    - Retourner dict avec clés : ticker, period, rsi (list float), macd_line, macd_signal, macd_histogram, bollinger_upper, bollinger_middle, bollinger_lower (chacun list float), dates (list str ISO)
    - Limiter à 100 points max (prendre les 100 derniers)

    Fonction `get_risk_metrics(ticker: str, period: str = "1y") -> dict` :
    - Télécharger historique 1y avec yfinance
    - Calculer rendements journaliers : returns = df["Close"].pct_change().dropna()
    - VaR 95% : np.percentile(returns, 5) * 100 (en %)
    - VaR 99% : np.percentile(returns, 1) * 100
    - Max drawdown : (df["Close"] / df["Close"].cummax() - 1).min() * 100
    - Volatilité annualisée : returns.std() * np.sqrt(252) * 100
    - Sharpe ratio : (returns.mean() * 252) / (returns.std() * np.sqrt(252)) si std != 0 sinon 0.0
    - Retourner dict avec : ticker, period, var_95, var_99, max_drawdown, volatility, sharpe_ratio (tous float arrondis 2 décimales)

    Pas de cache dans ce fichier — le cache est géré par le service ETF, ici on reste simple.
    Pas de dépendances circulaires avec yahoo_etf_service.
  </action>
  <verify>
    `cd backend && python -c "from services.analytics_service import get_indicators, get_risk_metrics; print(get_risk_metrics('AAPL'))"` retourne un dict avec var_95, sharpe_ratio etc.
  </verify>
