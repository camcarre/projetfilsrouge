<objective>
## Goal
Appliquer les 3 quick wins UI issus de l'audit UX/UI : unifier la navigation active (emerald desktop = mobile), établir un heading scale typographique, créer le composant DataRow, et corriger les contrastes dark mode.

## Purpose
L'audit a identifié une incohérence majeure : la nav desktop utilise neutral-200/80 (terne) alors que la nav mobile utilise emerald (vivant). Les données financières manquent de hiérarchie visuelle. Ces fixes améliorent la cohérence et la lisibilité sans toucher à l'architecture.

## Output
- Layout.tsx mis à jour (nav active desktop → emerald-50/700)
- tailwind.config.js étendu (heading scale h1/h2/h3)
- src/index.css : tabular-nums utilitaire + correction dark labels
- src/components/ui/DataRow.tsx créé (nouveau composant)
</objective>

    Créer le fichier src/components/ui/DataRow.tsx :

    ```tsx
    interface DataRowProps {
      label: string
      value: string | number
      variant?: 'default' | 'positive' | 'negative' | 'highlight'
    }

    const variantClasses: Record<NonNullable<DataRowProps['variant']>, string> = {
      default: 'text-neutral-900 dark:text-neutral-100',
      highlight: 'text-neutral-900 dark:text-neutral-100 font-bold',
      positive: 'text-emerald-600 dark:text-emerald-400',
      negative: 'text-red-600 dark:text-red-400',
    }

    export function DataRow({ label, value, variant = 'default' }: DataRowProps) {
      return (
        <div className="flex justify-between items-baseline gap-4">
          <span className="text-xs font-medium text-neutral-500 dark:text-neutral-300">
            {label}
          </span>
          <span className={`text-base font-semibold tabular-nums ${variantClasses[variant]}`}>
            {value}
          </span>
        </div>
      )
    }
    ```

    Utiliser des imports Preact si nécessaire (pas d'import React). Le composant est pur, pas de state ni d'effet.
    Exporter en named export uniquement.
