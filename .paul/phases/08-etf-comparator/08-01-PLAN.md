<objective>
## Goal
Ajouter un graphique multi-courbes dans la modale "Comparer une sélection" de la page ETF, affichant l'évolution normalisée (base 100) des ETF sélectionnés sur 3 mois.

## Purpose
Issue #9 (Comparateur d'ETFs — graphique multi-courbes). La modale de comparaison existe déjà (table de métriques) mais n'affiche aucune donnée visuelle d'évolution — l'utilisateur ne peut comparer que des chiffres statiques, pas les tendances.

## Output
- Nouveau composant `MultiLineChart` (SVG, jusqu'à 3 séries superposées, normalisées base 100)
- Intégration dans la modale de comparaison de `EtfPage.tsx` : fetch historique par ETF sélectionné, rendu du graphique, gestion loading/erreur par ligne
</objective>

<acceptance_criteria>

## AC-1: Affichage du graphique multi-courbes
```gherkin
Given l'utilisateur a sélectionné 2 ou 3 ETF dans la modale de comparaison
When la modale de comparaison est ouverte
Then un graphique affiche une courbe normalisée (base 100 au premier point) par ETF sélectionné, sur 3 mois, avec une légende indiquant le nom de chaque ETF
```

## AC-2: Résilience si un historique échoue
```gherkin
Given un des ETF sélectionnés n'a pas de données historiques disponibles (erreur API ou 404)
When le graphique se charge
Then les autres courbes s'affichent normalement et l'ETF en échec est signalé par un badge d'erreur inline, sans bloquer l'affichage du reste
```

## AC-3: Pas de graphique en dessous de 2 sélections
```gherkin
Given moins de 2 ETF sont sélectionnés dans la modale
When la modale de comparaison est ouverte
Then le graphique n'est pas rendu (le message "Aucune sélection" existant reste inchangé)
```

</acceptance_criteria>

  <name>Task 1: Créer le composant MultiLineChart</name>
  <files>src/components/ui/MultiLineChart.tsx</files>
  <action>
    Créer un composant SVG autonome, sur le modèle du style de `CombinedChart.tsx` (dimensions ~560x240, padding, grille horizontale légère, `currentColor` + classes Tailwind pour dark mode) mais supportant plusieurs séries nommées au lieu d'une seule.

    Props :
    ```ts
    interface Series {
      label: string
      data: { date: string, value: number }[]
      failed?: boolean
    }
    { series: Series[], className?: string, height?: number | string }
    ```

    Comportement :
    - Ignorer les séries `failed` ou vides pour le tracé, mais les garder dans la légende avec un badge d'erreur (texte rouge-600, ex: "Données indisponibles").
    - Normaliser chaque série valide en base 100 par rapport à sa première valeur : `value / data[0].value * 100`.
    - Calculer min/max sur l'ensemble des séries normalisées pour un axe Y commun.
    - Respecter la contrainte de palette du projet (neutral + emerald + red uniquement, voir PROJECT.md) : différencier les courbes par `strokeDasharray` (solide / tirets courts / tirets longs) plutôt que par teinte, toutes en `text-emerald-600 dark:text-emerald-400`, sauf si la performance totale de la série est négative auquel cas utiliser `text-red-600 dark:text-red-400`.
    - Légende sous le graphique : un badge par ETF (pastille + trait miniature reprenant le style de la ligne + nom + performance totale en %).
    - Pas de dépendance à GSAP ni à une lib de charting externe (pas de Recharts dans ce projet malgré la mention dans CLAUDE.md — rester cohérent avec l'existant).
    Éviter : ajouter une nouvelle couleur hors palette, ajouter une dépendance npm.
