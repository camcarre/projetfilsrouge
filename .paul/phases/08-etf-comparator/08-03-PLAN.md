<objective>
## Goal
Ajouter une matrice de corrélation (heatmap) entre les actifs du portefeuille de l'utilisateur, affichée dans la card "Graphiques interactifs" (actuellement un texte statique de remplissage) de la page Analyse.

## Purpose
Issue #12 (Corrélation entre actifs du portfolio). Note importante découverte durant la planification : la phase 08-02 du ROADMAP (VaR + Drawdown + Sharpe UI) est en réalité déjà livrée par la Phase 05 Plan 02 — `AnalysisPage.tsx` affiche déjà ces métriques (card "Risque", lignes ~414-477), branchées sur `GET /api/analyze/risk/{ticker}` réel. Ce plan couvre donc uniquement la partie corrélation, seule brique réellement manquante de la phase 08.

## Output
- Backend : `GET /api/analyze/correlation` — calcule la corrélation de Pearson entre les rendements quotidiens des actifs du portefeuille de l'utilisateur authentifié
- Frontend : composant `CorrelationHeatmap` (grille colorée, pas de nouvelle couleur hors palette neutral/emerald/red) branché dans la card "Graphiques interactifs" d'`AnalysisPage.tsx`
</objective>

<acceptance_criteria>

## AC-1: Endpoint de corrélation
```gherkin
Given un utilisateur authentifié possède au moins 2 actifs distincts dans son portefeuille
When il appelle GET /api/analyze/correlation?period=1y
Then la réponse contient la liste des tickers et une matrice N×N de coefficients de corrélation (Pearson, rendements quotidiens sur la période), avec 1.0 sur la diagonale
```

## AC-2: Portefeuille insuffisant
```gherkin
Given un utilisateur authentifié possède moins de 2 actifs distincts (ou aucun)
When il appelle GET /api/analyze/correlation
Then l'API retourne 400 avec un message explicite, et le frontend affiche un message informatif au lieu de la heatmap ("Ajoutez au moins 2 actifs à votre portefeuille pour voir leur corrélation")
```

## AC-3: Affichage de la heatmap
```gherkin
Given le portefeuille contient au moins 2 actifs et la corrélation a été calculée avec succès
When la page Analyse est affichée
Then la card "Graphiques interactifs" montre une grille N×N colorée (emerald = corrélation positive, red = corrélation négative, intensité proportionnelle à |coefficient|) avec les tickers en en-tête de ligne et colonne
```

    Dans `analytics_service.py`, ajouter `get_correlation_matrix(tickers: list[str], period: str = "1y") -> dict` sur le modèle de `get_risk_metrics` (lignes 136-174) :
    - Télécharger l'historique de tous les tickers en un seul appel `yf.download(tickers, period=period)["Close"]` (plus efficace que N appels séquentiels comme dans `etfs_compare`).

    - Calculer la matrice de corrélation via `.corr()` pandas (Pearson), arrondir à 2 décimales.
    - Retourner `{"tickers": [...], "matrix": [[...], ...], "period": period}`.

    Dans `main.py`, ajouter la route après `analyze_risk` (~ligne 744) :
    ```
    @app.get("/api/analyze/correlation", status_code=200)
    ```

    - Appeler `get_correlation_matrix`, gérer `ValueError` → 503 et `Exception` générique → 503 (même pattern que `analyze_risk`).
