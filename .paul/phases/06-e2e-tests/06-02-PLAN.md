<objective>
## Goal
Mettre en place un pipeline CI/CD GitHub Actions qui valide automatiquement lint + build + tests Playwright sur chaque PR vers main.

## Purpose
Sans CI, les bugs peuvent merger dans main sans détection. Ce pipeline garantit que chaque PR passe les mêmes vérifications que le dev local avant d'être mergée — essentiel en travail à 2.

## Output
- `.github/workflows/ci.yml` : workflow GitHub Actions avec 3 jobs (lint, build, playwright)
</objective>

<task type="auto">
  <name>Créer le workflow GitHub Actions CI</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Créer `.github/workflows/ci.yml` avec :

    - Trigger : `on: [pull_request]` vers `main` + `push` vers `main`
    - Node version : 20 (lts)
    - Python version : 3.11 (compatible avec le backend FastAPI)

    3 jobs séquentiels :

    **Job 1 — lint**
    - `working-directory: .` (racine frontend)
    - `npm ci`
    - `npm run lint`

    **Job 2 — build** (needs: lint)
    - `npm ci`
    - `npm run build`

    **Job 3 — playwright** (needs: build)
    - Install deps frontend : `npm ci`
    - Install deps backend : `cd backend && pip install -r requirements.txt`
    - Install Playwright browsers : `npx playwright install --with-deps chromium`
    - Démarrer backend en background : `cd backend && uvicorn main:app --port 8000 &`
    - Démarrer preview frontend en background : `npm run build && npx vite preview --port 4173 &`
    - Attendre que les serveurs soient prêts : `npx wait-on http://localhost:4173 http://localhost:8000/docs --timeout 30000`
    - Lancer les tests : `npx playwright test`
    - Upload artifact en cas d'échec : `uses: actions/upload-artifact@v4` sur `playwright-report/`

    Utiliser `continue-on-error: false` sur lint et build.
    Utiliser `continue-on-error: true` sur playwright (le backend peut échouer en CI sans .env).

    Avoid : installer toutes les dépendances Python système inutiles, utiliser des actions dépréciées (actions/checkout@v3 → v4).
  </action>
  <verify>
    - Fichier `.github/workflows/ci.yml` valide YAML
    - `gh workflow list` montre le workflow après push
    - Structure jobs lint → build → playwright visible
  </verify>
  <done>AC-1, AC-2, AC-3, AC-4 satisfaits : pipeline déclenché sur PR avec 3 jobs</done>
</task>
